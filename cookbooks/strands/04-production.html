Production & ElastiCache

Deploy Strands agents with Valkey session persistence on Amazon ElastiCache. Tested with ElastiCache Serverless Valkey 8.1 and ElastiCache Valkey 8.2.

Prerequisites

  • Completed Cookbooks 01 and 02
  • An ElastiCache for Valkey cluster (Serverless Valkey 8.1 or Valkey 8.2)
  • AWS credentials configured

Swap in Your ElastiCache Endpoint

The only change from local development is the connection parameters. ValkeySessionManager accepts any valkey.Valkey client — just swap in your ElastiCache endpoint:

import valkey
from strands import Agent
from strands_valkey_session_manager import ValkeySessionManager

# ElastiCache for Valkey endpoint
ELASTICACHE_HOST = "master.my-valkey-cluster.abc123.use1.cache.amazonaws.com"

# Production client with TLS and connection pooling
client = valkey.Valkey(
    host=ELASTICACHE_HOST,
    port=6379,
    ssl=True,
    ssl_cert_reqs="none",  # ElastiCache uses AWS-managed certs
    decode_responses=True,
    socket_connect_timeout=5,
    socket_timeout=5,
    retry_on_timeout=True,
    health_check_interval=30,
)

session_manager = ValkeySessionManager(
    session_id="user-42",
    client=client,
)

agent = Agent(
    system_prompt="You are a helpful assistant.",
    session_manager=session_manager,
)

VPC access: ElastiCache clusters are not publicly accessible. Your Lambda function, ECS task, or EC2 instance must be in the same VPC and security group as the cluster. For local development, use an SSM port-forwarding tunnel.

Connect from Your Laptop

To connect to ElastiCache from your laptop during development:

# Start the tunnel (requires a jump host EC2 instance in the same VPC)
aws ssm start-session \
  --target i-0594295b8314b037f \
  --document-name AWS-StartPortForwardingSessionToRemoteHost \
  --parameters '{
    "host":["master.my-valkey-cluster.abc123.use1.cache.amazonaws.com"],
    "portNumber":["6379"],
    "localPortNumber":["6379"]
  }' \
  --profile default \
  --region us-east-1

Then connect to localhost:6379 as normal — the tunnel forwards to ElastiCache.

Deploy to Lambda

Initialize the Valkey client outside the handler to reuse connections across warm invocations:

import os
import valkey
from strands import Agent
from strands_valkey_session_manager import ValkeySessionManager

# Module-level — reused across Lambda invocations
_client = valkey.Valkey(
    host=os.environ["ELASTICACHE_HOST"],
    port=6379,
    ssl=True,
    ssl_cert_reqs="none",
    decode_responses=True,
    socket_connect_timeout=3,
    socket_timeout=3,
)

def handler(event, context):
    session_id = event["session_id"]
    message = event["message"]

    session_manager = ValkeySessionManager(
        session_id=session_id,
        client=_client,
    )
    agent = Agent(session_manager=session_manager)
    response = str(agent(message))

    return {"statusCode": 200, "body": response}

Serve Multiple Users

For apps serving many users, create a helper that returns a session-scoped agent:

def get_agent(session_id: str, system_prompt: str = "You are a helpful assistant.") -> Agent:
    """Return a Strands agent with Valkey-backed session for the given session_id."""
    session_manager = ValkeySessionManager(
        session_id=session_id,
        client=_client,
    )
    return Agent(
        system_prompt=system_prompt,
        session_manager=session_manager,
    )

# Each user gets their own isolated session
agent_alice = get_agent("user-alice")
agent_bob = get_agent("user-bob")

agent_alice("I prefer concise answers.")
agent_bob("I'm working on a Python project.")

Keep Memory Usage in Check

Configure ElastiCache to use allkeys-lru so the least-recently-used sessions are evicted first when memory is full:

# Set via AWS CLI (one-time cluster configuration)
aws elasticache modify-replication-group \
  --replication-group-id my-valkey-cluster \
  --cache-parameter-group-name valkey8-allkeys-lru \
  --apply-immediately \
  --profile default \
  --region us-east-1

Check What's Active

def session_health(client: valkey.Valkey) -> dict:
    """Count active sessions and total message keys."""
    all_keys = client.keys("session:*")
    session_keys = [k for k in all_keys if k.count(":") == 1]
    msg_keys = [k for k in all_keys if ":message:" in k]
    return {
        "active_sessions": len(session_keys),
        "total_messages": len(msg_keys),
        "total_keys": len(all_keys),
        "valkey_ping": client.ping(),
    }

print(session_health(_client))
# {'active_sessions': 42, 'total_messages': 318, 'total_keys': 445, 'valkey_ping': True}
